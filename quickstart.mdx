---
title: "Quick start"
description: "Create your first project spec and generate an Xcode project in minutes"
---

<Steps>
  <Step title="Install XcodeGen">
    Follow the [installation guide](/installation) to install XcodeGen on your machine using Homebrew, Mint, Make, or Swift Package Manager.
  </Step>

  <Step title="Create a project spec">
    In the root of your project directory, create a file named `project.yml`. This file describes your targets, dependencies, and build settings.

    ```yaml
    name: MyProject
    options:
      bundleIdPrefix: com.myapp
    packages:
      Yams:
        url: https://github.com/jpsim/Yams
        from: 2.0.0
    targets:
      MyApp:
        type: application
        platform: iOS
        deploymentTarget: "10.0"
        sources: [MyApp]
        settings:
          configs:
            debug:
              CUSTOM_BUILD_SETTING: my_debug_value
            release:
              CUSTOM_BUILD_SETTING: my_release_value
        dependencies:
          - target: MyFramework
          - package: Yams
      MyFramework:
        type: framework
        platform: iOS
        sources: [MyFramework]
    ```

    See the [project spec reference](/reference/project-spec) for the full list of available options.
  </Step>

  <Step title="Generate the Xcode project">
    Run the following command from the directory containing your `project.yml`:

    ```bash
    xcodegen generate
    ```

    XcodeGen reads the spec, parses your source directories, and writes an `.xcodeproj` file.

    <Note>
      By default, the generated project is placed in the same directory as your `project.yml` file. Use the `--project` flag to output it to a different directory.
    </Note>

    <Tip>
      Pass the `--use-cache` flag to skip regeneration when nothing has changed since the last run. XcodeGen writes a cache file and compares it on subsequent runs, making it safe to call on every build without overhead.

      ```bash
      xcodegen generate --use-cache
      ```
    </Tip>
  </Step>

  <Step title="Open the project in Xcode">
    Open the generated `.xcodeproj` file in Xcode:

    ```bash
    open MyProject.xcodeproj
    ```

    All groups and file references reflect your directory structure on disk.
  </Step>

  <Step title="Add the generated project to .gitignore">
    Because the `.xcodeproj` can always be regenerated from the spec, you can safely exclude it from version control to eliminate merge conflicts:

    ```bash
    # .gitignore
    *.xcodeproj
    ```

    Commit your `project.yml` instead. Team members generate the project locally by running `xcodegen generate` after cloning.
  </Step>
</Steps>
