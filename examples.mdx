## Common Spec Patterns

The annotated examples below cover the most frequently used XcodeGen configurations.

### Multi-target app with a local framework

A typical setup where an iOS app links against a locally defined Swift framework, both sharing the same build settings.

```yaml
name: MyApp
options:
  bundleIdPrefix: com.example
  deploymentTarget:
    iOS: "16.0"

configs:
  Debug: debug
  Release: release

settings:
  base:
    SWIFT_VERSION: "5.9"
    IPHONEOS_DEPLOYMENT_TARGET: "16.0"

targets:
  # The main application target
  MyApp:
    type: application
    platform: iOS
    sources:
      - App
    dependencies:
      - target: MyFramework   # Link the local framework
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.example.myapp
        DEVELOPMENT_TEAM: XXXXXXXXXX

  # A reusable framework bundled alongside the app
  MyFramework:
    type: framework
    platform: iOS
    sources:
      - MyFramework
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.example.myframework

  # Unit tests for the framework
  MyFrameworkTests:
    type: bundle.unit-test
    platform: iOS
    sources:
      - MyFrameworkTests
    dependencies:
      - target: MyFramework
```

### App with Swift Package dependencies

Declaring both remote and local Swift Package Manager dependencies at the project level and linking them to targets.

```yaml
name: MyApp

packages:
  # Remote package pinned to a version range
  Alamofire:
    url: https://github.com/Alamofire/Alamofire
    from: "5.8.0"

  # Remote package pinned to an exact version
  SnapshotTesting:
    url: https://github.com/pointfreeco/swift-snapshot-testing
    exactVersion: "1.15.4"

  # Local package at a path relative to the project root
  MyLocalPackage:
    path: ../MyLocalPackage

targets:
  MyApp:
    type: application
    platform: iOS
    sources:
      - Sources
    dependencies:
      - package: Alamofire        # Links Alamofire product
      - package: MyLocalPackage   # Links the local package

  MyAppTests:
    type: bundle.unit-test
    platform: iOS
    sources:
      - Tests
    dependencies:
      - target: MyApp
      - package: SnapshotTesting
        product: SnapshotTesting  # Specify the product name when ambiguous
```

### Multi-platform framework

A framework that compiles for both iOS and macOS by leveraging `supportedDestinations`, sharing one target definition.

```yaml
name: CrossPlatformKit
options:
  bundleIdPrefix: com.example

targets:
  CrossPlatformKit:
    type: framework
    # Use 'auto' platform with supportedDestinations for multi-platform targets
    platform: auto
    supportedDestinations:
      - iOS
      - macOS
    sources:
      - Sources/CrossPlatformKit
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.example.crossplatformkit
        SWIFT_VERSION: "5.9"
        MACOSX_DEPLOYMENT_TARGET: "13.0"
        IPHONEOS_DEPLOYMENT_TARGET: "16.0"

  CrossPlatformKitTests:
    type: bundle.unit-test
    platform: auto
    supportedDestinations:
      - iOS
      - macOS
    sources:
      - Tests/CrossPlatformKitTests
    dependencies:
      - target: CrossPlatformKit
```
